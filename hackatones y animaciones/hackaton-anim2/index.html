Baja por la página. Las tarjetas y los textos deben aparecer con animación cuando entran al viewport. Usa IntersectionObserver — lo mismo que viste en la página modelo para los contadores.

LAS TARJETAS
TE ESPERAN

01

OBSERVER

IntersectionObserver detecta cuándo un elemento entra al área visible del navegador sin necesidad de calcular nada con scrollY.

automático
02

THRESHOLD

El threshold indica qué porcentaje del elemento debe ser visible para disparar la animación. 0.2 significa el 20%.

0.2
03

UNOBSERVE

Después de agregar la clase .visible, llamamos unobserve para que el observer deje de escuchar ese elemento. Solo aparece una vez.

una vez
04

CSS HACE
EL TRABAJO

La animación está definida en CSS. JS solo agrega la clase. Separación de responsabilidades: JS maneja lógica, CSS maneja visual.

separación
05

DELAY
ESCALONADO

Cada tarjeta tiene un transition-delay distinto en CSS. Así aunque JS les agrega la clase al mismo tiempo, aparecen una tras otra.

nth-child
06

A LA VEZ

querySelectorAll devuelve todas las tarjetas. forEach las recorre para que el observer las vigile a todas con un solo bloque de código.
